# ANTIGRAVITY 專案指導原則 (Project Guidelines)

> [!NOTE]
> 本文件是本專案的「最高指導原則（Source of Truth）」。Agent 在每次開工或開啟新對話時，必須優先閱讀此文件，並嚴格遵守以下約定。

---

## 📖 專案基本資訊 (Project Info)
*   **專案名稱**：Excel 巨集與民國曆日期計算工具箱
*   **專案目標**：提供一套完整的 Excel VBA 巨集庫，專門處理台灣中華民國曆（民國年）日期的解析、計算、格式化與轉換，確保行政與財務系統的日期運算精確無誤。
*   **目標客群 / 使用場景**：台灣學校行政人員、公務機關資訊人員、金融機構財務人員，用於自動化處理含民國年日期的 Excel 報表。

---

## 🛠 技術棧與環境要求 (Tech Stack & Environment)
*   **開發語言**：Excel VBA (Visual Basic for Applications)
*   **核心框架/庫**：Excel Object Model, VBA Standard Library
*   **套件管理工具**：無 (手動匯入 `.bas` 或 `.cls` 模組檔案)
*   **環境需求**：Microsoft Excel 2010 或更新版本 (支援 VBA 7.1)
*   **環境變數檔**：若巨集需與外部 API 互動，使用 `.env` 或 Registry 管理私鑰，嚴禁將私鑰寫死在 VBA 代碼中。

---

## 📁 專案目錄結構規範 (Directory Structure)
> 為了保持程式碼整潔，請將檔案建立在以下約定的目錄中：
*   `src/`：放置核心 VBA 程式碼模組檔案（`.bas` 標準模組、`.cls` 類別模組、`.frm` 表單設計）。
*   `tests/`：放置 VBA 測試巨集（用於執行單元測試的測試子程序）。
*   `docs/`：放置專案相關說明文件與 API 文件。
*   `templates/`：放置含巨集的 Excel 範本檔（`.xlsm` 或 `.xltm`）。
*   `.agents/`：放置專案特定的 AI 規則與協同代理配置。

---

## 🛡️ 開發守則與代碼風格 (Development & Coding Standards)
*   **命名規範**：變數必須使用匈牙利命名法（例如：`wsTarget`、`strRocDate`），函式與子程序使用大駝峰命名（PascalCase）。
*   **註解規範**：所有 VBA 函式與複雜邏輯必須使用「繁體中文」撰寫詳細註解。
*   **民國曆處理**：所有日期計算皆須遵循 `.agents/AGENTS.md` 中規定的「中華民國曆日期計算規範」，嚴禁寫死魔術數字 `1911` 或直接對民國字串做數值加減。
*   **Git 分支管理**：
    *   開發分支：請在 `feature/` 相關分支上開發。
    *   主要分支：`main` 分支為生產環境，需通過審查合併。

---

## ⚠️ 絕對禁止事項 (Never Do)
1.  **嚴禁寫死敏感資訊**：禁止將 API 金鑰、資料庫密碼或機敏個資直接寫死在 VBA 程式碼或表單中。
2.  **嚴禁直接對民國字串做數值加減來計算日期**：例如將 `"1120315" + 1` 作為翌日，必須使用 VBA 內建的 `DateAdd` 進行日期運算。
3.  **嚴禁覆蓋既有功能**：在修改 VBA 模組前，必須確認不會破壞已存在的正常功能，必要時先執行測試程序。
